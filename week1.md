# Git 개념 정리

## Git이란?
파일의 변경 사항을 추적하고 여러 사용자 간 협업을 가능하게 하는 **분산 버전 관리 시스템**.

## Working Directory
실제 사용자가 작업하는 공간으로, 현재 변경된 파일들이 있는 영역.

## Staging Area
커밋을 하기 전에 파일 변경 사항을 임시로 저장하는 공간 (`git add`를 하면 여기에 저장됨).

## Local Repository
로컬(개인 PC)에서 Git이 관리하는 저장소로, 커밋된 변경 사항들이 보관됨.

## .gitignore란?
Git이 추적하지 않을 파일이나 폴더를 지정하는 설정 파일  
(예: 로그 파일, 빌드 파일 등).

## git add, git commit, git push란?

- `git add <파일>` → 변경된 파일을 **Staging Area**로 올림
- `git commit -m "메시지"` → Staging Area에 있는 변경 사항을 **로컬 저장소에 기록**
- `git push origin <브랜치명>` → 로컬 저장소의 변경 사항을 **원격 저장소에 업로드**

## 브랜치(Branch)란?
독립적인 작업을 위한 별도의 개발 흐름을 의미하며, 여러 작업을 동시에 진행할 수 있도록 돕는 기능.

## 이슈(Issue)란?
버그 수정, 기능 추가 등 작업할 내용을 정리하고 관리하는 GitHub의 기능 (프로젝트 관리용).

## PR(Pull Request)이란?
원격 저장소에서 변경된 내용을 다른 브랜치(예: `main`)에 병합 요청하는 과정.
