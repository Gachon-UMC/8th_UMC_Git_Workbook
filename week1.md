# 오늘 공부한 내용
## Git은 왜 사용하는가?
- 여러 사람과 협업할 때 버전 관리를 위해
## Repository
- 로컬 레포지토리, 원격 레포지토리가 있음
- 로컬은 내가 작업하는 개인적인 공간, 원격은 내가 협업을 위해 공유하는 공간
## add
- 내가 작업한 파일들을 작업대(stage) 위에 올림
- add .을 통해 모든 파일을 올리기도, add (파일 이름)으로 특정 파일만 올리기도 가능
## commit
- 내가 stage에 올린 작업물을 저장
## push
- 내가 commit한 작업물을 원격 레포지토리에 반영시킴
## branch
- 독립적인 작업 공간을 하나 새로 팜
- 한 branch에서 작업하는 내용은 다른 branch에 영향을 주지 않음
- 만들 땐 git branch (브랜치 이름), 다른 branch로 이동할 땐 git switch (이동할 브랜치 이름)
## log
- git log를 통해 나의 커밋 기록 확인 가능
## Issue
- 기능 브랜치를 나누기 전에 팀원들과의 작업 현황, 이슈들을 공유함
- 자세한 작성 방법은 프로젝트 컨벤션마다 다름
## PR
- 특정 브랜치와 다른 브랜치를 병합하는 요청을 보냄
- Reviewers들을 설정하여 내 코드를 리뷰해줬으면 좋겠는 사람을 설정 가능
- PR없이 바로 타 브랜치와의 병합은 매우 위험!
