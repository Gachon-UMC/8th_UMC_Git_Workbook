# Git, Github 개념

1. **Git**: 파일의 변경 사항 추적하고, 협업을 위한 '분산 버전 관리 시스템'

2. **Github**: 원격 저장소(remote)

3. **commit**: 변경사항이 생긴 상태의 체크포인트 저장

4. **add**: 수정된 파일을 stage(선택적 커밋할 수 있도록)하는 것  
   * `add` → `commit` → `push` 순으로 진행되겠군.

5. **clone**: `git clone (레포지토리 주소)`

6. **.gitignore**: 워킹 디렉토리 내에서 추적되지 않아야 하는 파일 목록 저장

7. **branch**: 독립적인 작업 영역  
   * `git branch (브랜치 이름)`  
   * `git switch (브랜치 이름)`

8. **merge**: `git merge (병합할 브랜치)`

9. **git log**: 커밋 기록 확인
