# 📌 GitHub 1주차 학습 정리

## 1. Git을 활용한 효율적인 버전 관리
- **Git**은 분산 버전 관리 시스템(DVCS) 중 하나로, 효율적인 버전 관리가 가능하다.
- `.git`을 활용하여 코드의 변경 사항을 추적하고 협업할 수 있다.

## 2. Staged 개념의 명확한 이해
- 기존에는 `commit` 전에 `add`를 실행하는 것만 알았지만, 이를 통해 파일을 **staged 상태**로 만들고 `commit`을 하면 **modified** 상태로 변하는 원리를 배웠다.
- **staged**와 **non-staged** 파일의 개념을 명확하게 이해함.

## 3. 로컬 & 원격 레포지토리 개념 정리
- **로컬 레포 (Local Repository)** → 내 컴퓨터에서 직접 수정하는 공간
- **원격 레포 (Remote Repository)** → GitHub에 업로드된 최종 저장소

## 4. `.gitignore` 활용
- `.gitignore` 사이트를 통해 불필요한 파일을 쉽게 관리 가능!  
  (예: `node_modules`, `*.log`, `.env` 파일 등)

## 5. Branch & Pull Request (PR) 워크플로우
- **바로 `main` 브랜치에 push하면 안됨!**  
  - 새로운 **branch**를 만들어 작업 후,  
  - `main` 브랜치로 **Pull Request(PR)** 제출!  
- 아직 PR을 직접 사용해보진 않았지만, 협업하면서 꼭 경험해보고 싶다.

## 6. GitHub Issue 기능
- `Issue` 기능을 활용하면 프로젝트 내 **작업을 시각적으로 관리**할 수 있다.
- **Assignees(담당자 지정), 제목, 내용**을 정리하여 문제를 명확하게 공유 가능.
- Git Flow 방식을 활용하면 팀원들과 효율적으로 협업 가능!

## 7. 코드 리뷰 (Code Review)
- **PR을 할 때 Reviewer(리뷰어)를 지정**하여 코드 리뷰를 진행하는 방식 학습.
- 단순히 코드만 올리는 것이 아니라, 피드백을 주고받으며 코드 품질을 향상시키는 과정이 중요함.
