# Git
Git = 협업을 위해 필요한 분산 버전 관리 시스템
- 파일의 변경 사항 추적 가능
- 여러 사용자 간에 파일들의 작업들 조율 가능

# 파일 관리 방식
## 나의 작업 공간
Working Directory = 현재 내가 작업하고 있는 공간 (변경 사항 대부분 추적 가능)
- .gitignore로 추적 X, 원격 레포 X

## Staging Area
= 변경 사항 반영하는 공간
- 변경 없음(unmodified)
- 변경됨(modified)
- 스테이지됨(staged)
    = 커밋을 위한 준비단계 (선택적으로 파일 커밋 위해)

## branch
- checkout
    = Git 2.23 이전에 사용.
    = 브랜치 이동 + 파일 복원
- switch
    = 브랜치 이동만
    ( 더 권장 )

## 커밋 기록 확인법
- git log
    - 커밋 해시 (커밋 구분 위한 고유 ID)
    - 커밋 생성자
    - 커밋 일시
    - 커밋 메시지
- HEAD
    = 내 브랜치의 가장 최신 주소
