# 1주차 과제 정리

---

## Git이란 무엇이고 왜 사용해야 할까요?

- Git은 **파일의 변경 사항을 추적하고**,  
  **여러 사용자가 작업 내용을 조율할 수 있게 해주는 분산 버전 관리 시스템**입니다.

---

### 1. 파일의 변경 사항을 추적

- 파일을 여러 개로 따로 저장하지 않고도 Git이 수정 기록을 저장해줌
- 특정 시점으로 언제든지 돌아갈 수 있음 (`버전 관리`)
- 변경 이력은 로그 형태로 관리됨

---

### 2. 여러 사용자 간 작업 조율

- Git은 원격 저장소(GitHub)를 통해 팀원이 **같은 작업을 공유하고 협업**할 수 있도록 도와줍니다.
- 한 파일을 여러 명이 동시에 작업할 수 있고, **충돌 없이 병합**할 수 있도록 도와줍니다.

---

### 3. 분산 버전 관리 시스템

- Git은 원격 저장소(GitHub)의 모든 기록을 **개별 사용자의 로컬 저장소로 복사**해서 사용
- 모든 개발자가 독립적으로 작업하고, 이후 병합할 수 있음

---

## Git 내부 구조

### 1. Working Directory (작업 공간)

- 실제 사용자가 작업하는 폴더
- `.gitignore` 파일을 통해 특정 파일 추적을 제외할 수 있음

### 2. Staging Area (커밋 대기 공간)

- `git add` 명령으로 **변경된 파일을 커밋 전에 준비**하는 공간
- 전체가 아닌 일부 파일만 선택적으로 커밋할 수 있음

### 3. Local Repository (로컬 저장소)

- `git commit`을 통해 실제 변경 기록을 남기는 저장소
- 이후 `git push`를 통해 원격(GitHub)으로 업로드 가능

---

## Git 기본 명령어

```bash
git status                # 현재 상태 확인
git add .                # 모든 변경 파일 staging
git add 파일명           # 특정 파일만 staging
git commit -m "메시지"  # 커밋 기록 남기기
git push origin 브랜치명 # 원격 저장소로 업로드
